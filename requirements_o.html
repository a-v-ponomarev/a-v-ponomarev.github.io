Требования к содержанию отчета

Для задачи динамического программирования

  1. Постановка задачи.
  2. Формализация задачи - что обозначают переменные, как через эти переменные записывается функциональное уравнение ДП.
  3. Результат: принимаемое на каждом шаге управление, итоговый выигрыш.
  4. Приложение - исходные тексты программы (Python, GNU Octave, C++, C, Java).

Для задач линейного программирования

  1. Постановка задачи
  2. Формализация задачи - что обозначают переменные, как через эти переменные выражаются целевая функция и ограничения, перечисленные в условии задачи. Математическая запись задачи линейного программирования.

...решаемых с помощью Python

  1. Описание задачи с помощью средств выбранной библиотеки (CVXOPT, PuLP).
  2. Решение задачи (значения переменных, при которых достигается максимум/минимум целевой функции, само значение целевой функции в точке максимума/минимума). Интерпретация решения в терминах постановки задачи.
  3. Исходные данные для анализа чувствительности и их интерпретация (необходимость выполнения этого пункта зависит от варианта, согласовать с преподавателем).
  4. Исследование решения для ответа на вопросы, указанные в варианте задания.

...решаемых с помощью GNU Octave

  1. Перевод математической записи задачи в таблицу, показывающую, какие параметры будут переданы функции glpk.
  2. Вызов glpk, включая определение всех матриц, передаваемых glpk. Матрицы, возвращаемые этой функцей.
  3. Решение задачи (значения переменных, при которых достигается максимум/минимум целевой функции, само значение целевой функции в точке максимума/минимума). Интерпретация решения в терминах постановки задачи.
  4. Исходные данные для анализа чувствительности и их интерпретация (необходимость выполнения этого пункта зависит от варианта, согласовать с преподавателем).
  5. Исследование решения для ответа на вопросы, указанные в варианте задания.

...решаемых с помощью GLPK

  1. Описание задачи на языке GNU MathProg (обязательно разделение на модель и данные).
  2. Решение задачи (значения переменных, при которых достигается максимум/минимум целевой функции, само значение целевой функции в точке максимума/минимума). Интерпретация решения в терминах постановки задачи.
  3. Исходные данные для анализа чувствительности и их интерпретация (необходимость выполнения этого пункта зависит от варианта, согласовать с преподавателем).
  4. Исследование решения для ответа на вопросы, указанные в варианте задания.

Требования к оформлению отчета

"Живой" пример отчета с определенными стилями можно взять здесь. При использовании шаблона, пожалуйста, старайтесь не пользоваться функциями форматирования, а применяйте к фрагментам текста соответствующий стиль ("Заголовок 1", "Заголовок 2", "Обычный", "Программа" и пр.).

Чеклист для защиты

Пожалуйста, перед сдачей убедитесь, что отчет содержит содержит:
  • актуальное задание на курсовую работу (назначенный вам вариант);
  • объяснение смысла всех используемых переменных и указание единиц измерения для них;
  • пояснение способа формирования целевой функции;
  • пояснение того, каким образом получены ограничения;
  • пояснение принципа решения (что взяли, куда загрузили, куда смотрели и пр.);
  • интерпретацию результатов в терминах исходной задачи (фразы "максимальное значение ЦФ достигается при x равном 14" мало).
